## 五、统一状态码定义

### 新增 AttendanceStatusEnum.java
**路径**: `zcst-manage/src/main/java/com/zcst/manage/constant/AttendanceStatusEnum.java`

**状态码定义**:
```java
public static final String NORMAL = "0";        // 正常
public static final String LATE = "1";          // 迟到
public static final String EARLY_LEAVE = "2";   // 早退
public static final String ABSENCE = "3";       // 缺勤
public static final String NOT_CHECKED = "0";   // 未打卡（用于值班表）
public static final String CHECKED = "1";       // 已打卡（用于值班表）
```

**使用情况**:
- `AttendanceRecordServiceImpl.checkIn()`: 使用 `NORMAL`, `LATE`
- `AttendanceRecordServiceImpl.checkOut()`: 使用 `EARLY_LEAVE`, `LATE`, `NORMAL`
- `AttendanceStatisticsServiceImpl.calculateMonthlyAttendance()`: 使用 `NORMAL`, `LATE`, `EARLY_LEAVE`

## 十、注意事项

1. **关联查询性能**: VO 查询使用了 LEFT JOIN，如果数据量大，建议添加索引
2. **状态码统一**: 所有状态判断都应使用 `AttendanceStatusEnum` 常量，避免使用魔法值
3. **事务控制**: 打卡和签退操作已添加 `@Transactional` 注解，确保数据一致性
4. **空值处理**: VO 查询可能返回 null（如学生已删除），前端需要做空值处理

---

## 十一、后续优化建议

1. **添加缓存**: 对于频繁查询的学生信息和场馆信息，可以考虑添加 Redis 缓存
