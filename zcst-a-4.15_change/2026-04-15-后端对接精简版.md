# 2026-04-15 后端对接精简版（个人中心）

## 1. 对接范围

- 学生个人中心信息保存
- 教务系统登录状态查询
- 教务系统账号密码登录并保存
- 头像上传（已联通，仅确认字段）

## 2. 接口清单

### 2.1 保存学生扩展信息

- URL: `PUT /manage/student/info`
- 请求体:

```json
{
  "venueId": 1,
  "collegeName": "计算机学院",
  "majorName": "软件工程"
}
```

- 后端要求:
- 支持接收 `collegeName`、`majorName`、`venueId`
- 落库更新并返回标准结构

### 2.2 查询教务系统登录状态（新增）

- URL: `GET /manage/student/academic-auth/status`
- 响应示例:

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "loggedIn": true
  }
}
```

### 2.3 教务系统登录并保存（新增）

- URL: `POST /manage/student/academic-auth/login`
- 请求体:

```json
{
  "account": "20260001",
  "password": "******",
  "syncCloud": true
}
```

- 成功响应示例:

```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "loginSuccess": true
  }
}
```

- 失败响应示例:

```json
{
  "code": 500,
  "msg": "账号或密码错误",
  "data": {
    "loginSuccess": false
  }
}
```

## 3. 前端判定逻辑（请按此返回）

- 登录成功判定: `code === 200` 且 `data.loginSuccess !== false`
- 其余情况均按失败处理，前端提示: `当前账号密码错误，登录失败`
- 登录成功后按钮由 `登录` 变为 `重新登录`

## 4. 验收标准

- 修改学院/专业/值班地点后点击保存，刷新页面数据不丢失
- 教务系统首次进入显示 `登录`
- 输入正确账号密码点击登录后，返回成功并显示 `重新登录`
- 输入错误账号密码点击登录后，提示 `当前账号密码错误，登录失败`
- 头像上传接口字段保持 `avatarfile`，请求类型为 `multipart/form-data`
