## 3. 后端需要修改/提供的内容

### 3.1 学生信息保存接口字段支持

接口：`PUT /manage/student/info`

当前前端会提交：

```json
{
  "venueId": 1,
  "collegeName": "计算机学院",
  "majorName": "软件工程"
}
```

后端需确认：
- DTO/VO/实体是否接收 `collegeName`、`majorName`
- Service/Mapper 是否执行更新
- 返回标准 `code/msg/data` 结构

### 3.2 教务系统登录状态查询接口（新增）

- URL：`GET /manage/student/academic-auth/status`
- 用途：初始化个人中心按钮文案（登录/重新登录）
- 建议返回：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "loggedIn": true
  }
}
```

### 3.3 教务系统登录并保存接口（新增）

- URL：`POST /manage/student/academic-auth/login`
- 请求体：

```json
{
  "account": "20260001",
  "password": "******",
  "syncCloud": true
}
```

- 前端判定规则：
  - 成功：`code === 200` 且 `data.loginSuccess !== false`
  - 失败：其他情况均提示 `当前账号密码错误，登录失败`

- 建议成功返回：

```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "loginSuccess": true
  }
}
```

- 建议失败返回：

```json
{
  "code": 500,
  "msg": "账号或密码错误",
  "data": {
    "loginSuccess": false
  }
}
```

## 4. 接口调用关系说明

### 4.1 个人中心保存

- 触发点：点击“保存资料”
- 调用顺序：
  1. `PUT /system/user/profile`（手机号、邮箱、性别、场馆等）
  2. 学生端额外调用 `PUT /manage/student/info`（学院、专业、场馆）

### 4.2 教务系统登录

- 页面加载（学生端）：`GET /manage/student/academic-auth/status`
- 点击“登录/重新登录”打开弹窗
- 弹窗点击“登录”：`POST /manage/student/academic-auth/login`
- 返回成功后：按钮文案从 `登录` 切换为 `重新登录`

### 4.3 修改头像

- 弹窗点击“保存头像”：`POST /system/user/profile/avatar`（multipart/form-data）
- 字段名：`avatarfile`
