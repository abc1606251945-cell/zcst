# 教务系统登录接口缺口记录

- 日期: 2026-04-15
- 功能名: 个人中心-教务系统登录
- 优先级: 高
- 负责人: 后端同学（待指派）
- 前端调用位置:
  - `src/views/system/user/profile/userInfo.vue`
  - `src/api/manage/student.js`

## 缺失接口 1: 查询教务系统登录状态
- 接口名称: `getAcademicAuthStatus`
- URL: `GET /manage/student/academic-auth/status`
- 请求参数: 无
- 期望响应:

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "loggedIn": true
  }
}
```

- 业务背景:
  - 个人中心右侧“教务系统”按钮需要根据状态展示“登录/重新登录”。

## 缺失接口 2: 教务系统登录并保存账号信息
- 接口名称: `loginAcademicAuth`
- URL: `POST /manage/student/academic-auth/login`
- 请求体:

```json
{
  "account": "20260001",
  "password": "******",
  "syncCloud": true
}
```

- 期望响应（登录成功）:

```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "loginSuccess": true
  }
}
```

- 期望响应（登录失败）:

```json
{
  "code": 500,
  "msg": "账号或密码错误",
  "data": {
    "loginSuccess": false
  }
}
```

- 业务背景:
  - 弹窗内点击“登录”后，前端需要等待后端登录结果；
  - 失败时前端固定提示“当前账号密码错误，登录失败”；
  - 成功后个人中心按钮从“登录”切换为“重新登录”。

## 风险提示
- 当前后端未就绪时，前端无法持久化教务系统登录状态，刷新页面会按未登录展示。
