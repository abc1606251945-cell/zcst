# 微信小程序学生端接口补齐说明（后端 zcst-b）
更新时间：2026-04-11

## 1. 概述与对前端需求文档的回应

针对前端同学提供的《2026-04-10 小程序学生端后端补充接口详细说明》，后端已全面完成了相关接口的开发和改造工作。以下为逐条回应与落实情况：

- **回应 3.1 首页值班表“按场馆+按周查看”能力不足**：
  - `GET /manage/dutySchedule/currentDuty` 接口用于获取当前时刻可签到的值班记录（不再信任/使用前端传入的 studentId，始终以当前登录用户为准）。
  - 针对完整的周视图需求，后端提供 `GET /manage/dutySchedule/student/week` 接口：
    - 强制按“当前登录学生 + 学生绑定场馆”取数（不信任前端 venueId）。
    - 支持返回 `recordId` 与 `canCheckOut`，方便前端统一签退协议。

- **回应 3.2 值班管理页仍是本地存储**：
  - 后端已新增 `/manage/duty/application` 接口组，将请假（leave）与调班/替班（swap）统一封装。前端现在可以直接调用接口提交、查询列表、查看详情及撤销申请。

- **回应 3.3 场馆页“可用人员”仍是本地数据**：
  - 后端已新增 `GET /manage/duty/free/student/list` 接口。支持按日期、时段（如 `1-2节`）及场馆查询真实的可用人员（已排除有课和已有值班冲突的学生）。

- **回应 3.4 通知中心仍是本地数据**：
  - 针对学生端，后端专门设计了 `student_notice` 表，并提供了专属的 `/manage/student/notice` 接口组，支持获取通知列表、单条/全部已读、未读数统计。

- **回应 3.5 签退接口缺失**：
  - 后端现已完善 `POST /manage/attendance/record/checkOut` 接口。
    - 推荐学生端优先传 `recordId` 进行签退（统一协议、幂等更明确）。
    - 兼容只传 `dutyId`：后端会自动根据“当前登录用户 + dutyId”定位未签退记录并签退。

- **回应 4.6 字典与低权限读取接口**：
  - 场馆简易列表：后端已提供 `GET /manage/venue/simple` 接口，仅返回 `venueId` 和 `venueName`。
  - 专业列表：后端已新增 `GET /manage/major/list` 接口，供学生端注册及资料页使用。

- **回应 5. 权限与安全建议**：
  - 所有学生端接口均已实现“**不信任前端传入的 studentId**”的原则，统一从当前登录用户的 Token 中提取学号，确保数据隔离和安全性。
  - 学生/管理员已做后端分权兜底：
    - 学生账号调用管理端列表（如 `/manage/dutySchedule/list`）将返回 `403`。
    - 管理端接口除 `@PreAuthorize` 外，还会在业务层/控制层做数据域二次校验（防误配权限绕过）。

- **回应 8. 联调验收点**：
  - 各个接口已支持按场馆正确过滤。
  - 防重复签退、防非法撤销等业务冲突均通过统一的业务错误码返回（`409`）。
  - 为防止国际化用户（如时区非东八区）的设备解析时间字符串发生偏移，后端在关键时间字段上额外补充了毫秒时间戳（`*Ts` 字段），**建议前端优先使用时间戳进行渲染**。

## 3. 详细接口清单与对接指南

### 3.1 首页周值班 (对应 P0 需求)

#### 3.1.1 获取学生周值班数据
- **方法**：`GET`
- **路径**：`/manage/dutySchedule/student/week`
- **鉴权**：需要登录 token（学生账号）
- **Query参数**：
  - `weekStart`：`yyyy-MM-dd`（可选；不传默认本周周一）
  - `venueId`：`int`（可选；仅允许等于学生绑定场馆；不传默认学生绑定场馆；传入其他场馆将返回空数组）
- **返回数据 (`data` 数组)**：
  - `dutyId`：值班ID
  - `studentId`、`studentName`：学号、姓名
  - `venueId`、`venueName`：场馆ID、场馆名称
  - `startTime` / `endTime`：格式 `yyyy-MM-dd HH:mm:ss`
  - `startTimeTs` / `endTimeTs`：毫秒时间戳（**防时区偏移推荐使用**）
  - `attendanceStatus`：考勤状态码（0正常 1迟到 2早退 3缺勤；为空表示未产生考勤）
  - `recordId`：考勤记录ID（已打卡则有值；用于前端签退）
  - `checkInTime` / `checkOutTime`：考勤打卡/签退时间（无则为空）
  - `canCheckIn`：布尔值（后端计算：当前时间在 start/end 之间且未打卡）
  - `canCheckOut`：布尔值（后端计算：已打卡且未签退）

### 3.2 考勤签退 (对应 P0 需求)

#### 3.2.1 签退
- **方法**：`POST`
- **路径**：`/manage/attendance/record/checkOut`
- **鉴权**：需要登录 token（学生账号）
- **Query参数（学生端用法）**：
  - `recordId`：`long`（推荐；考勤记录ID，必须属于当前登录用户）
  - `dutyId`：`int`（可选；兼容模式：仅传 dutyId 时，后端用“当前登录用户 + dutyId”定位未签退记录并签退）
- **说明**：
  - 如果传入 `recordId`，后端将校验该记录归属当前用户，否则返回 `403`。
  - 如果重复签退、未打卡签退、或当前无可签退记录，将返回业务冲突码 `409`。

### 3.3 值班申请 (请假/替班) (对应 P1 需求)

#### 3.3.1 提交申请
- **方法**：`POST`
- **路径**：`/manage/duty/application`
- **鉴权**：需要登录 token
- **Body参数 (JSON)**：
  - `applicationType`：`leave` 或 `swap`（必填）
  - `dutyId`：`int`（必填；必须属于当前登录学生）
  - `reason`：`string`（必填）
  - `relatedStudentId`：`string`（仅 swap 必填；替班人学号）
  - `leaveType`：`string`（仅 leave 可选；0病假 1事假 2其他）
  - `proofImage`：`string`（仅 leave 可选；证明材料 URL）

#### 3.3.2 我的申请列表
- **方法**：`GET`
- **路径**：`/manage/duty/application/my`
- **鉴权**：需要登录 token
- **Query参数**：
  - `status`：`pending` | `approved` | `rejected` | `cancelled`（可选；不传表示全部）
- **返回数据 (`data` 数组)**：
  - 统一返回 `applicationType`、`id`、`dutyId`、`venueId`、`venueName`、`startTime`、`endTime` 及对应时间戳。
  - `status`：统一映射后的状态字符串（`pending`/`approved`/`rejected`/`cancelled`）。
  - `statusCode`：原始数据库状态码。

#### 3.3.3 撤销申请
- **方法**：`PUT`
- **路径**：`/manage/duty/application/cancel/{id}`
- **鉴权**：需要登录 token
- **Query参数**：
  - `applicationType`：`leave` | `swap`（必填）
  - `reason`：`string`（可选）
- **规则限制**：
  - 仅允许在待审批状态下撤销。一旦管理员审批完成，接口将拦截撤销并返回业务冲突码 `409`。

### 3.4 场馆可用人员查询 (对应 P1 需求)

#### 3.4.1 获取空闲学生列表
- **方法**：`GET`
- **路径**：`/manage/duty/free/student/list`
- **鉴权**：需要登录 token
- **Query参数**：
  - `dutyDate`：`yyyy-MM-dd`（必填）
  - `period`：如 `1-2节`、`3-4节` 或 `HH:mm-HH:mm`（必填）
  - `venueId`：`int`（可选；不传则返回全部场馆的汇总可用人员）
- **返回数据 (`data` 数组)**：
  - 包含 `studentId`、`studentName`、`venueId`、`venueName`。
  - 提供 `startTimeTs` / `endTimeTs`（毫秒时间戳），对应 period 落在 dutyDate 当天的确切起止时间。

### 3.5 学生通知中心 (对应 P1 需求)

后端提供完整的已读/未读状态管理：
- **通知列表**：`GET /manage/student/notice/list?readFlag=0/1`
- **单条已读**：`PUT /manage/student/notice/read/{noticeId}`
- **全部已读**：`PUT /manage/student/notice/readAll`
- **未读数**：`GET /manage/student/notice/unread/count`

说明：
- 单条已读会校验通知归属；如果传入他人 `noticeId`，返回 `403`。

### 3.6 字典与低权限读取 (对应 P2 需求)

- **场馆简易列表**：`GET /manage/venue/simple`
  - 仅返回 `venueId` 和 `venueName`。
- **专业字典列表**：`GET /manage/major/list`
  - 返回 `majorId`、`majorName`、`collegeId`。

## 4. 权限与错误码约定（学生端/管理端分权后）

### 4.1 学生端权限标识（后端内置）

学生账号默认拥有以下权限标识（用于 `@PreAuthorize` 拦截），无需依赖 sys_menu 配置：
- `manage:dutySchedule:studentWeek`
- `manage:dutyApplication:student`
- `manage:studentNotice:student`
- `manage:attendance:student`

### 4.2 错误码语义（响应体 code）

后端统一使用响应体 `code` 表达错误语义：
- `400`：参数错误
- `403`：无权限
- `409`：业务冲突（重复签退/重复打卡/非法撤销等）
