# 文件上传功能文档

## 1. 功能概述

文件上传功能用于支持学生上传课表照片到阿里云 OSS，采用 x-file-storage 框架实现，具有自动分类、序号持久化、双模式支持等特性。

### 1.1 核心特性

- **自动分类存储**：按上传日期自动分类（年/月/日目录结构）
- **序号持久化**：通过查询 OSS 当天文件数量实现序号递增，重启不重置
- **双模式支持**：支持登录模式（自动获取学号）和测试模式（手动输入）
- **学号隔离**：学号仅用于日志记录，不包含在文件路径中
- **无随机后缀**：保持原始文件扩展名，不添加随机字符串

## 3. API 接口说明

### 3.1 课表照片上传接口

#### 接口信息
- **接口地址**：`POST /upload/schedule`
- **接口说明**：上传课表照片到阿里云 OSS
- **权限要求**：支持匿名访问（@Anonymous）

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| file | MultipartFile | 是 | 上传的图片文件（支持 jpg、png、gif 等格式） |
| studentId | String | 否 | 学号（已登录时自动获取，未登录时需要传递） |

#### 请求示例

**cURL 命令**：
```bash
curl -X POST http://localhost:8080/upload/schedule \
  -F "file=@timetable.jpg" \
  -F "studentId=20260321"
```

**前端 FormData 示例**：
```javascript
const formData = new FormData();
formData.append('file', fileInput.files[0]);
formData.append('studentId', '20260321');

fetch('http://localhost:8080/upload/schedule', {
  method: 'POST',
  body: formData
});
```

#### 响应格式

**成功响应**：
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": "https://zcsst-student-timetable.oss-cn-beijing.aliyuncs.com/timetable/schedule/2026/03/29/202603290001.jpg"
}
```

**失败响应**：
```json
{
  "code": 500,
  "msg": "上传失败：课表照片上传失败：文件类型不支持",
  "data": null
}
```

#### 状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 上传成功 |
| 400 | 请求参数错误（如文件为空） |
| 401 | 未登录且未传递 studentId 参数 |
| 500 | 上传失败（文件类型不支持、OSS 服务异常等） |

## 4. 文件命名规则

### 4.1 文件路径结构

```
timetable/                              # 根目录（base-path）
  schedule/                             # 课表目录
    {年}/{月}/{日}/                     # 日期目录
      {yyyyMMdd}{序号}.{扩展名}         # 文件名
```

### 4.2 完整路径示例

```
https://zcsst-student-timetable.oss-cn-beijing.aliyuncs.com/timetable/schedule/2026/03/29/202603290001.jpg

路径解析：
timetable/                    # base-path（application.yml 配置）
  schedule/                   # 课表目录
    2026/03/29/              # 日期（年/月/日）
      202603290001.jpg       # 文件名（日期 +4 位序号）
```

### 4.3 文件命名规则详解

| 组成部分 | 格式 | 长度 | 示例 | 说明 |
|---------|------|------|------|------|
| 日期 | yyyyMMdd | 8 位 | 20260329 | 上传日期的 8 位数字 |
| 序号 | 0001-9999 | 4 位 | 0001 | 4 位数字序号，从 0001 开始递增 |
| 扩展名 | 原文件扩展名 | 可变 | jpg | 保持原始文件扩展名 |

**文件名格式**：`{日期}{序号}.{扩展名}`

**示例**：
- `202603290001.jpg` - 2026 年 3 月 29 日第 1 个上传的文件
- `202603290002.png` - 2026 年 3 月 29 日第 2 个上传的文件
- `202603300001.jpg` - 2026 年 3 月 30 日第 1 个上传的文件（新的一天，序号重置）

### 4.4 序号生成规则

#### 实现原理

1. **查询 OSS 当天文件**：上传时查询 OSS 上当天（{年}/{月}/{日}）已有的文件
2. **提取已用序号**：从文件名中提取已使用的序号（使用正则表达式）
3. **分配下一个序号**：从 1 开始递增，找到第一个未使用的序号
4. **持久化机制**：通过查询 OSS 实现序号持久化，服务重启后不会重置

#### 序号查询逻辑

```java
private Set<Integer> queryTodayFileSequences(String datePath) {
    Set<Integer> sequences = new HashSet<>();
    
    // 使用阿里云 OSS ListObjects API 查询当天文件
    OSS ossClient = createOSSClient();
    String prefix = "schedule/" + datePath + "/";
    
    ObjectListing listing = ossClient.listObjects(
        bucketName, 
        prefix
    );
    
    // 从文件名中提取序号
    for (OSSObjectSummary object : listing.getObjectSummaries()) {
        String filename = object.getKey();
        // 文件名格式：yyyyMMddXXXX.ext
        // 提取第 9-12 位作为序号
        if (filename.length() >= 16) {
            String seqStr = filename.substring(8, 12);
            try {
                int seq = Integer.parseInt(seqStr);
                sequences.add(seq);
            } catch (NumberFormatException e) {
                // 忽略格式错误的文件名
            }
        }
    }
    
    return sequences;
}
```

#### 序号分配逻辑

```java
private String generateSequentialFilename(String datePath, String originalFilename) {
    // 查询当天已使用的序号
    Set<Integer> existingSequences = queryTodayFileSequences(datePath);
    
    // 从 1 开始递增，找到第一个未使用的序号
    int sequence = 1;
    while (existingSequences.contains(sequence)) {
        sequence++;
    }
    
    // 生成文件名
    String extension = getExtension(originalFilename);
    String dateStr = LocalDate.now().format(FILE_DATE_FORMATTER);
    return dateStr + String.format("%04d", sequence) + "." + extension;
}
```

## 6. 使用模式

### 6.1 登录模式（正式环境）

**适用场景**：用户已登录系统的正式环境

**工作流程**：
1. 用户登录系统，获取 JWT Token
2. 前端在请求中携带 Token
3. 后端从 Token 中自动提取学号（SecurityUtils.getUsername()）
4. 学号仅用于日志记录，不包含在文件路径中

**示例代码**：
```java
@Anonymous
@PostMapping("/schedule")
public AjaxResult uploadSchedule(
        @RequestParam("file") MultipartFile file,
        @RequestParam(value = "studentId", required = false) String studentId) {
    
    // 如果没有传递 studentId，尝试从登录信息中获取
    if (studentId == null || studentId.trim().isEmpty()) {
        try {
            studentId = SecurityUtils.getUsername();
            log.info("从登录信息获取学号：{}", studentId);
        } catch (Exception e) {
            return AjaxResult.error("未登录，请传递 studentId 参数或先登录系统");
        }
    }
    
    // 验证文件
    if (file.isEmpty()) {
        return AjaxResult.error("上传文件不能为空");
    }
    
    // 上传文件
    String url = fileUploadService.uploadScheduleImage(file, studentId);
    return AjaxResult.success("上传成功", url);
}
```

```java
package com.zcst.upload.service.impl;

import com.zcst.upload.service.FileUploadService;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.dromara.x.file.storage.core.FileInfo;
import org.dromara.x.file.storage.core.FileStorageService;
import org.springframework.stereotype.Service;
import org.springframework.web.multipart.MultipartFile;

import java.time.LocalDate;
import java.time.format.DateTimeFormatter;

/**
 * 文件上传服务实现类
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class FileUploadServiceImpl implements FileUploadService {

    private final FileStorageService fileStorageService;
    
    private static final DateTimeFormatter DATE_FORMATTER = 
        DateTimeFormatter.ofPattern("yyyy/MM/dd");
    private static final DateTimeFormatter FILE_DATE_FORMATTER = 
        DateTimeFormatter.ofPattern("yyyyMMdd");

    @Override
    public String uploadScheduleImage(MultipartFile file, String studentId) {
        try {
            log.info("开始上传课表照片 - 文件名：{}, 类型：{}, 大小：{}, 学号：{}", 
                     file.getOriginalFilename(), 
                     file.getContentType(),
                     file.getSize(),
                     studentId);
            
            // 生成日期路径
            String datePath = LocalDate.now().format(DATE_FORMATTER);
            
            // 生成顺序文件名
            String filename = generateSequentialFilename(datePath, file.getOriginalFilename());
            
            // basePath 已经在 application.yml 中配置为 timetable/
            String fullPath = "schedule/" + datePath + "/";
            
            log.info("上传路径：{}", fullPath);
            log.info("保存文件名：{}", filename);
            
            // setPath 设置目录路径（以/结尾），setSaveFilename 设置文件名
            FileInfo fileInfo = fileStorageService.of(file)
                    .setPath(fullPath)
                    .setSaveFilename(filename)
                    .upload();
            
            if (fileInfo == null) {
                throw new RuntimeException("文件上传失败");
            }
            
            log.info("课表照片上传成功：{}", fileInfo.getUrl());
            return fileInfo.getUrl();
        } catch (Exception e) {
            log.error("课表照片上传失败", e);
            throw new RuntimeException("课表照片上传失败：" + e.getMessage(), e);
        }
    }

        return dateStr + String.format("%04d", sequence) + "." + extension;
    }

    /**
     * 查询当天已使用的序号
     */
    private Set<Integer> queryTodayFileSequences(String datePath) {
        Set<Integer> sequences = new HashSet<>();
        
        try {
            // 使用阿里云 OSS ListObjects API 查询当天文件
            // 注意：这里需要使用阿里云 OSS SDK
            // 具体实现参考 4.4 节
        } catch (Exception e) {
            log.error("查询当天文件序号失败", e);
        }
        
        return sequences;
    }

    /**
     * 获取文件扩展名
     */
    private String getExtension(String originalFilename) {
        if (originalFilename == null || originalFilename.isEmpty()) {
            return "";
        }
        
        int lastDotIndex = originalFilename.lastIndexOf(".");
        if (lastDotIndex == -1 || lastDotIndex == originalFilename.length() - 1) {
```

## 8. 常见问题

### 8.1 文件上传失败：不支持此文件

**问题现象**：
```
org.dromara.x.file.storage.core.exception.FileStorageRuntimeException: 不支持此文件
```

**可能原因**：
1. 启动类未添加 `@EnableFileStorage` 注解
2. x-file-storage 配置不正确
3. 依赖配置不完整

**解决方案**：
1. 确认启动类添加了 `@EnableFileStorage` 注解
2. 检查 application.yml 中的 OSS 配置是否正确
3. 确认 pom.xml 中包含了 x-file-storage-spring 依赖

### 8.2 文件名重复

**问题现象**：
```
202603290001.jpg202603290001.jpg
```

**可能原因**：
- `setPath()` 已经包含了文件名，又用 `setSaveFilename()` 设置了一次

**解决方案**：
```java
// setPath 只设置目录路径（以/结尾）
String fullPath = "schedule/" + datePath + "/";
fileStorageService.of(file)
        .setPath(fullPath)
        .setSaveFilename(filename)  // 只设置文件名
        .upload();
```

### 8.3 路径重复

**问题现象**：
```
https://...aliyuncs.com/timetable/timetable/schedule/...
```

**可能原因**：
- basePath 已经是 `timetable/`，代码中又拼接了一次

**解决方案**：
```java
// basePath 已经在 application.yml 中配置为 timetable/
// 代码中不需要再拼接 timetable/
String fullPath = "schedule/" + datePath + "/";
```

### 8.4 序号不连续

**问题现象**：
- 同一天上传的文件序号不连续（如 0001、0003、0005）

**可能原因**：
- 并发上传时，多个请求同时查询 OSS，导致序号分配冲突

**解决方案**：
- 当前实现已能处理大部分并发场景
- 如需严格连续，可考虑使用 Redis 原子计数器

## 10. 部署说明

### 10.1 环境要求

- Java 17+
- Spring Boot 4.0.3+
- 阿里云 OSS 账号（AccessKey、SecretKey、Bucket）

### 10.2 部署步骤

1. **配置 OSS 信息**
   - 修改 `application.yml` 中的 OSS 配置
   - 替换 AccessKey、SecretKey、Endpoint、Bucket 名称

2. **编译项目**
   ```bash
   mvn clean install -DskipTests
   ```

3. **启动服务**
   ```bash
   java -jar zcst-admin/target/zcst-admin.jar
   ```

4. **验证部署**
   - 访问 Swagger UI：`http://localhost:8080/swagger-ui.html`
   - 使用测试页面上传文件，验证功能正常

### 10.3 生产环境注意事项

1. **安全配置**
   - 使用环境变量或配置中心管理 AccessKey 和 SecretKey
   - 不要将敏感信息提交到代码仓库

2. **性能优化**
   - 配置 OSS 上传超时时间
   - 考虑使用 OSS 传输加速功能

3. **监控告警**
   - 监控 OSS 存储空间使用量
   - 监控上传失败率
