# 课表相关表结构汇总（供 Python 按表结构返回数据）

本文档整理 `d:\zcst\zcst-b\sql\zcst.sql` 中与“学生课表/课程入库”直接相关的表结构，包含建表语句与字段说明。

## 1. student_schedule（学生课表）

用途：存储学生每一节课的上课时间段，后端导入课表后会写入本表。

### 建表语句

```sql
DROP TABLE IF EXISTS `student_schedule`;
CREATE TABLE `student_schedule` (
  `schedule_id` int NOT NULL AUTO_INCREMENT COMMENT '课表ID',
  `student_id` varchar(20) NOT NULL COMMENT '学号',
  `course_name` varchar(100) NOT NULL COMMENT '??????',
  `location` varchar(100) DEFAULT NULL COMMENT '??????',
  `start_time` datetime NOT NULL COMMENT '课程开始时间（格式：2026-03-14 08:00:00）',
  `end_time` datetime NOT NULL COMMENT '课程结束时间（格式：2026-03-14 09:45:00）',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`schedule_id`),
  KEY `idx_student_id` (`student_id`),
  KEY `idx_time_range` (`start_time`,`end_time`),
  KEY `idx_location` (`location`)
) ENGINE=InnoDB AUTO_INCREMENT=97 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='学生课表';
```

### 字段说明

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---:|---|
| schedule_id | int | 否 | 主键自增；导入时无需提供 |
| student_id | varchar(20) | 是 | 学号（导入维度） |
| course_name | varchar(100) | 是 | 课程名称 |
| location | varchar(100) | 否 | 上课地点 |
| start_time | datetime | 是 | 课程开始时间（具体日期+时间） |
| end_time | datetime | 是 | 课程结束时间（必须大于 start_time） |
| created_at | datetime | 否 | 默认当前时间；导入时无需提供 |
| updated_at | datetime | 否 | 默认当前时间；导入时无需提供 |

## 2. student（学生）

用途：学生基础信息表，课表通过 `student_id` 关联到学生。

### 建表语句

```sql
DROP TABLE IF EXISTS `student`;
CREATE TABLE `student` (
  `student_id` varchar(20) NOT NULL COMMENT '学号',
  `user_id` bigint DEFAULT NULL COMMENT '关联 sys_user 表 ID',
  `name` varchar(50) NOT NULL COMMENT '姓名',
  `gender` char(1) NOT NULL COMMENT '性别（男/女）',
  `major_id` int NOT NULL COMMENT '专业ID',
  `grade` varchar(20) NOT NULL COMMENT '年级',
  `class_name` varchar(50) DEFAULT NULL COMMENT '班级',
  `phone` varchar(11) NOT NULL COMMENT '手机号码',
  `venue_id` int DEFAULT NULL COMMENT '场馆 ID',
  `password` varchar(255) NOT NULL DEFAULT '123456' COMMENT '密码',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`student_id`),
  UNIQUE KEY `uk_user_id` (`user_id`),
  KEY `idx_grade` (`grade`),
  KEY `idx_major_id` (`major_id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_venue_id` (`venue_id`),
  KEY `idx_phone` (`phone`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='学生表';
```

### 字段说明

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---:|---|
| student_id | varchar(20) | 是 | 学号（主键） |
| user_id | bigint | 否 | 关联系统用户（通常由触发器/注册逻辑生成） |
| name | varchar(50) | 是 | 姓名 |
| gender | char(1) | 是 | 性别 |
| major_id | int | 是 | 专业 ID（关联 major.major_id） |
| grade | varchar(20) | 是 | 年级 |
| class_name | varchar(50) | 否 | 班级名称 |
| phone | varchar(11) | 是 | 手机号 |
| venue_id | int | 否 | 场馆 ID（关联 venue.venue_id） |
| password | varchar(255) | 是 | 密码（与 sys_user 同步） |
| created_at | datetime | 否 | 默认当前时间 |
| updated_at | datetime | 否 | 自动更新时间 |

## 3. major（专业）

用途：专业维表，学生通过 `major_id` 关联。

### 建表语句

```sql
DROP TABLE IF EXISTS `major`;
CREATE TABLE `major` (
  `major_id` int NOT NULL AUTO_INCREMENT COMMENT '专业ID',
  `major_name` varchar(100) NOT NULL COMMENT '专业名称',
  `college_id` int NOT NULL COMMENT '所属学院ID',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`major_id`),
  UNIQUE KEY `uk_college_major` (`college_id`,`major_name`),
  KEY `idx_college_id` (`college_id`)
) ENGINE=InnoDB AUTO_INCREMENT=7 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='专业表';
```

### 字段说明

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---:|---|
| major_id | int | 否 | 主键自增 |
| major_name | varchar(100) | 是 | 专业名称 |
| college_id | int | 是 | 学院 ID（关联 college.college_id） |
| created_at | datetime | 否 | 默认当前时间 |
| updated_at | datetime | 否 | 自动更新时间 |

## Python 返回数据时的建议（按入库最小集）

若 Python 仅负责“课表解析导入”，通常只需要按 `student_schedule` 组织结果并返回：
- `student_id`
- `course_name`
- `location`（可空）
- `start_time` / `end_time`（datetime，具体日期时间，且 end_time > start_time）
