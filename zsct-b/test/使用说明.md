# 文件上传功能 - 使用说明

## 📋 功能说明

上传课表照片接口支持**两种使用模式**：

### 模式一：已登录系统（正式环境）

- **场景**：用户已登录系统
- **学号获取**：自动从登录 token 中获取
- **前端操作**：不需要传递 studentId 参数
- **示例**：正式的前端页面、已登录的 APP

### 模式二：未登录系统（测试环境）

- **场景**：用户未登录系统
- **学号获取**：需要手动传递 studentId 参数
- **前端操作**：必须传递 studentId 参数
- **示例**：测试页面、Postman 测试、命令行脚本

## 🔧 后端实现

### Controller 代码

```java
@PostMapping("/schedule")
public AjaxResult uploadSchedule(
        @RequestParam("file") MultipartFile file,
        @RequestParam(value = "studentId", required = false) String studentId) {
    
    // 如果没有传递 studentId，尝试从登录信息中获取
    if (studentId == null || studentId.trim().isEmpty()) {
        try {
            studentId = SecurityUtils.getUsername();
        } catch (Exception e) {
            return AjaxResult.error("未登录，请传递 studentId 参数或先登录系统");
        }
    }
    
    // 验证和上传逻辑...
}
```

## 📝 测试方法

### 方法一：测试页面（未登录）

**适用场景**：快速测试，无需登录

**步骤**：
1. 打开 `test/upload-test.html`
2. **输入学号**（必填）
3. 选择图片文件
4. 点击"上传"按钮

**请求参数**：
```javascript
{
    file: File,        // 图片文件
    studentId: "20260321"  // 学号（必填）
}
```

---

### 方法二：Postman 测试（未登录）

**适用场景**：API 调试

**步骤**：
1. 创建 POST 请求：`http://localhost:8081/upload/schedule`
2. Body 选择 `form-data`
3. 添加字段：
   - `file` (Type: File) - 选择图片
   - `studentId` (Type: Text) - 输入学号（如：20260321）
4. 发送请求

**响应示例**：
```json
{
    "code": 200,
    "msg": "上传成功",
    "data": "https://zcsst-student-timetable.oss-cn-beijing.aliyuncs.com/timetable/schedule/20260321/2026/03/29/202603290001.jpg"
}
```

---

### 方法三：正式前端页面（已登录）

**适用场景**：生产环境

**步骤**：
1. 用户登录系统（获取 token）
2. 前端在请求头或 cookie 中携带 token
3. 上传图片，**不需要传递 studentId**
4. 后端自动从 token 获取学号

**请求示例**：
```javascript
// 方式 1：使用 Cookie
fetch('/upload/schedule', {
    method: 'POST',
    credentials: 'include',  // 携带 cookie
    body: formData
});

// 方式 2：使用 Header
fetch('/upload/schedule', {
    method: 'POST',
    headers: {
        'Authorization': 'Bearer YOUR_TOKEN'
    },
    body: formData
});
```

**请求参数**：
```javascript
{
    file: File  // 只需要图片文件
    // 不需要 studentId，后端自动获取
}
```

---

### 方法四：命令行测试（未登录）

**适用场景**：自动化脚本

**步骤**：
1. 编辑 `test/test-upload.bat`
2. 设置图片路径和学号
3. 运行脚本

**cURL 命令示例**：
```bash
curl -X POST http://localhost:8081/upload/schedule \
  -F "file=@/path/to/image.jpg" \
  -F "studentId=20260321"
```

## 📁 文件路径示例

无论哪种模式，文件都会按照以下规则存储：

```
timetable/schedule/{学号}/{年}/{月}/{日}/{yyyyMMdd}{序号}.jpg

模式一（已登录）：
timetable/schedule/20260321/2026/03/29/202603290001.jpg

模式二（未登录）：
timetable/schedule/20260101/2026/03/29/202603290001.jpg
```

## ⚠️ 注意事项

### 1. 参数验证

- **已登录**：studentId 可选（为空时自动获取）
- **未登录**：studentId 必填

### 2. 错误处理

**未登录且未传递 studentId**：
```json
{
    "code": 500,
    "msg": "未登录，请传递 studentId 参数或先登录系统"
}
```

**studentId 为空**：
```json
{
    "code": 500,
    "msg": "学生 ID 不能为空"
}
```

### 3. 安全性

- **正式环境**：必须使用 token 认证，防止学号伪造
- **测试环境**：仅用于开发测试，不要用于生产

## 🎯 使用建议

### 开发阶段

1. **使用测试页面**：快速验证上传功能
2. **使用 Postman**：调试 API 接口
3. **传递 studentId**：简单直接

### 生产环境

1. **必须登录**：确保用户身份真实
2. **自动获取学号**：防止参数伪造
3. **Token 认证**：使用 cookie 或 header 传递 token
